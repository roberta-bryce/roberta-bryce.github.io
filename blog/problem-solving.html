23/09/22 Sprint 4 Problem Solving
A time you were blocked on a simple problem:

The Built-in Methods ‘Where Are You?’ Kata had two different problems that I’m sure were simple but for some reason my brain couldn’t compute. First I was unable to figure out the name of the object, and second I couldn’t get anything working outside of the function.

I actually never figured out the name of the object. I checked console.log(arrayOfObjects) in the whereAreYou_test.js file and tried the names I thought could apply cycling through them. With no luck. I thought I needed the name of the object as you need to call the whereAreYou function outside of itself to use the find method, and once I left the whereAreYou function arrayOfObjects was not defined…

This also leads to my second problem of I couldn’t code outside of my functions, it would cause the test program to have a wobble and interfere with other functions. This meant I couldn’t call the function while using the find method and couldn’t use the object name even if I did work it out. I tried rewriting my code with small alterations but nothing was working.

So rather than solving those two problems separately I combined them as the solution needed was ‘how do I get everything working within the function?’ This led to a lot of googling and a lot of youtube tutorials until I found an example of using the find method in one line using a fat arrow, which I updated with the correct details for the object:

const result = arrayOfObjects.find(({ hiding }) => hiding === 'Scooby');

Worked a treat! Felt very satisfying to get it working. Throughout the process I was annoyed that my issue involved working outside a function as normally that wouldn’t be an issue, also this looked deceptively easy so I felt a bit bad tripping up on this question. That was all backburner though, for the most part my emotions were focused on reaching the finish line of the Katas.

Overall I learnt that perseverance works. I knew the solution was out there, it just took time to get there, and along the way my understanding of the find method improved so it’s no loss.

 An elegantly solved a problem:

The Kata of FizzBuzz was my smoothest coding experience this Sprint.

I kept the problem explanation as pseudocode at the top to refer to. Even with the way the problem was phrased referenced using a conditional. So I used my favourite if/else.

So I coded it up looking good, but still bringing up an error. I rubber duckie-d it and yeah I felt really good about my maths so what was happening? Turned to my bff google and immediately found out the issue I was using divide rather than remainder - forgot I was talking to a computer and not a maths teacher. Once I changed my divides over to remainders it FizzBuzzed like a champion.

Throughout the process I felt pretty keyed in, so I didn’t really keep tabs of emotions.

On this one I learnt to respect my code for the dumb robot that she is and got a bit more comfortable with remainders.
